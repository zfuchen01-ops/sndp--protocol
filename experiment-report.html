SBDP 位于 IP 层。UE→gNB 是 IP 包，gNB↔gNB (ISL) 还是 IP 包，gNB→GW 也是 IP 包。只要包在 IP 层转发，SBDP 就寄生在上面逐跳更新瓶颈。

三种链路，SBDP 都要处理

链路标准名通信双方SBDP 做什么
接入链路Uu 口 / 5G NR用户 UE ↔ 星上 gNBgNB 读接入带宽, 写入 SBDP 报头
星间链路ISL / 激光或微波卫星 gNB ↔ 卫星 gNB每跳 gNB 读 ISL 带宽, min 更新瓶颈字段
馈电链路Feeder / SGL星上 gNB ↔ 地面网关读馈电带宽, min 更新, 到达地面站

SBDP 不是独立的一层。它寄生在 IP 层 (IPv6 Hop-by-Hop Option)，跨层读链路层带宽、写网络层字段。这不是"违反分层"，是跨层优化——卫星链路变化太快，纯端到端猜测跟不上。

协议是什么

SBDP (Satellite Backhaul Discovery Protocol)：卫星网络主动、持续地向用户推送"你到互联网的回传瓶颈是多少 Mbps"，用户无需探测。

核心机制 — 逐跳 min 更新

Satellite receives SBDP packet:
  1. PeekHeader  → read backhaul_bw
  2. RemoveHeader
  3. checkBw = min(incoming_link_bw, outgoing_link_bw)
  4. if checkBw < backhaul_bw:  backhaul_bw = checkBw   ★ N2 ① 瓶颈带宽
  5. latency += outgoing_link_latency                       ★ N2 ④ 时延累加
  6. jitter = max(jitter, outgoing_link_jitter)            ★ N2 ④ 抖动取max
  7. pkt_loss = 1-(1-pkt_loss)*(1-outgoing_link_loss)      ★ N2 ④ 丢包累积
  8. avail_cap = min(avail_cap, outgoing_link_bw)          ★ N2 ⑤ 可用容量
  9. hop_count++, ttl--
 10. AddHeader   → re-serialize with new CRC
 11. Forward to next hop

N2 接口 — 星载gNB↔星载路由器 五项信息

SBDP 承载 N2 接口的全部五项信息，让 gNB 能够全面了解承载网状态：

N2 信息项说明SBDP 字段类型逐跳处理
① 瓶颈带宽路由器暴露链路带宽backhaul_capacity (Header 固定字段, float32)承载网min(出口带宽, 传入值)
② 瓶颈位置精确指向拥塞段BOTTLENECK_LINK (TLV 0x0003, string)承载网瓶颈变化时更新
③ 路径追踪暴露用户路径属性PATH_TRACE (TLV 0x0004, string)承载网每跳追加 "→下一跳"
④ 链路 QoS链路 QoS 暴露至 gNBLINK_LATENCY (TLV 0x000A, float32 ms)
LINK_JITTER (TLV 0x000B, float32 ms)
LINK_PKT_LOSS (TLV 0x000C, float32 比率)		承载网时延: 逐跳累加 Σ
抖动: 逐跳取 max
丢包: 累积 1-∏(1-pᵢ)
⑤ 业务属性按用户/业务类别区分TRAFFIC_CLASS (TLV 0x000D, string: "realtime"/"bulk"/"best-effort")
USER_PRIORITY (TLV 0x000E, uint8: 0=best-effort ~ 7=network-control)		接入侧不修改 (gNB 设置)
N2 请求-响应模式：SBDP 定义了 CAPACITY_REQ (gNB→路由器查询) 和 CAPACITY_ACK (路由器→gNB应答) 消息类型。SBDPEngine.build_n2_response() 可构造包含全部五项信息的完整 N2 应答报文 (TTL=1, 单跳)。引擎当前默认使用 Push 模式 (CAPACITY_ADV)，REQ/ACK 模式可按需启用。

N3 接口 — gNB 间接入协同

N3 是星载 gNB 之间的逻辑接口，通过 ISL 承载。SBDP 用两种方式实现 N3：轻量捎带（日常状态共享）和迁移握手（群切换协同）。

N3 功能说明SBDP 实现方式
gNB 负载共享基站负载在 gNB 间扩散GNB_LOAD (TLV 0x0011, float32 0.0~1.0)捎带在 ADV 里广播给覆盖区内所有 UE + gNB
信道质量共享空口信道质量在 gNB 间扩散CHANNEL_QUALITY (TLV 0x0012, float32 SINR dB)捎带在 ADV 里
群切换协同迁移多基站协同低开销接入迁移CAPACITY_MIGRATE (0x05) / CAPACITY_CONFIRM (0x06)
HandoverCoordinator 类		gNB 间点对点握手
N3 两种工作模式：

① 捎带模式 (日常)：卫星在 ADV 里顺便带上 GNB_LOAD 和 CHANNEL_QUALITY，覆盖区内所有 UE 和其他 gNB 都能收到。UE 综合回传 + 负载 + 信道做评分排名。gNB 也能互相感知对方状态。

② 迁移模式 (群切换时)：当 gNB 过载或大批 UE 需要同时切走时，源 gNB 不是让 UE 各自盲抢，而是：查邻居表按评分排序 → 向最优候选星发 MIGRATE → 收到 CONFIRM（接/拒+替代建议）→ 分配 UE → 完成切换。 
# 群切换迁移流程 (HandoverCoordinator)
源gNB (SAT-B, 负载90%):
  coord = HandoverCoordinator("SAT-B")
  coord.update_neighbor("SAT-A", backhaul=500, GnBState(0.6, 15dB))
  coord.update_neighbor("SAT-C", backhaul=600, GnBState(0.4, 12dB))
  # 5个UE需要切走
  assignments = coord.decide_migration(["UE-1".."UE-5"], required_bw=50M)
  # → UE-1,UE-2,UE-3→SAT-C, UE-4,UE-5→SAT-A
  # 对每个目标星发 MIGRATE 请求确认

目标gNB (SAT-C):
  conf = coord.process_migrate_request(req, my_state=GnBState(0.4,...))
  # → 确认接UE-1,2,3, 预留150M, 拒绝0个

NS-3 验证 — 4 个实验

所有实验使用真实 NS-3 网络栈——卫星通过 UDP socket 发送 SBDP 二进制报文（16B 固定头 + TLV 链 + CRC-16），用户通过 socket 接收解析，PCAP 可逐字节验证。

实验 1 — 协议二进制证明

4 节点串行链，SBDP 报文逐跳更新。这是协议最底层的正确性保证——证明二进制格式、CRC、逐跳 min 在真实网络包中正确。

拓扑User-1(150M)→SAT-A(600M)→SAT-B(350M)→Server(1000M)
协议流程User 发探测包(backhaul=INF) → SAT-A Peek→Remove→min(INF,150)=150→更新→重算CRC→转发 → SAT-B min(150,350)=150不变→转发 → Server 收到 backhaul=150M ✓
验证min(150,600,350,1000) = 150M。PCAP 三段链路抓包，逐字节验证。

实验 2 — 多星覆盖接入切换

3 卫星 + 2 地面站 + 3 用户，80s。每颗卫星推送给覆盖区内所有用户，用户维护回传表，回传低于阈值自动切换。合并了原实验 2（切换）和实验 6（多星覆盖）。

实验 4 — N3 gNB 协同群切换对比

5 颗卫星（SAT-B 过载，12 UE 需切走）+ 真实 UDP 消息。对比「无 N3：UE 各自盲抢」和「有 N3：Coordinator 统一分配」。gNB 间通过 UDP/IP/ISL 发送 MIGRATE/CONFIRM 消息。

候选星回传负载SINR评分
SAT-E350M30%18dB146
SAT-C600M40%12dB144
SAT-A500M60%15dB100
SAT-D400M85%10dB20（跳过）
实测结果（NS-3 输出）：

无 N3：12 个 UE 全部涌向回传最高的 SAT-C(600M) → 只能接 5 个 → 7 个被拒 → 反复重试 → 140 次拒绝，285 条信令，7 个 UE 仍然失败

有 N3：Coordinator 按综合评分分配：5→SAT-E, 4→SAT-C, 3→SAT-A → 3 次 MIGRATE + 3 次 CONFIRM → 12/12 全部一次成功，0 拒绝，6 条消息

★ 关键发现：SAT-E 回传只有 350M 但综合评分(146)反而超过 SAT-C 的 600M(144)——因为 SAT-E 负载低(30%)、SINR 高(18dB)。仅按回传选星会忽略这些因素，N3 的多维评分纠正了这个问题。SAT-D(85%负载)被正确跳过。6 条 MIGRATE/CONFIRM 消息全部经真实 UDP socket/ISL 传输，PCAP 可验证。
